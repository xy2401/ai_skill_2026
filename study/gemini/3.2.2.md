# 3.2.2 手写数字识别系统交互流程设计解析

## 1. 实验目标
本实验旨在构建一个完整的手写数字识别应用。通过加载 MNIST 训练得到的 ONNX 模型，对输入图片进行深度学习推理。实验重点展示了针对灰度图像的特定预处理技术，以及模型在端侧（Inference Runtime）的部署流程。

## 2. 核心技术要点

### 2.1 图像预处理的“四个步骤”
在手写数字识别中，预处理决定了识别的成败：
1.  **色彩空间转换 (L 模式)**：MNIST 模型要求输入为单通道灰度图（黑白），必须将 RGB 彩图转换。
2.  **尺寸对齐 (28x28)**：神经网络输入层的大小是固定的。
3.  **数据类型归一化**：将 `uint8` (0-255) 转换为 `float32` (0.0-1.0)，这符合神经元激活函数的输入要求。
4.  **维度补全 (NCHW)**：模型要求 4 维张量输入（BatchSize, Channel, Height, Width）。由于单张图没有 Batch 和 Channel 维度，需使用 `np.expand_dims` 进行两次补全。

### 2.2 ONNX 推理机制
- **InferenceSession**：核心引擎，负责解析计算图。
- **Input Dictionary**：推理时需要构建一个 Python 字典，Key 必须与模型的输入节点名称（通常为 `Input3` 或 `data`）严格匹配。

### 2.3 结果后处理
模型输出是一个长度为 10 的向量，代表 0 到 9 每个数字的得分。通过 `np.argmax` 函数，我们选取其中得分最高的那一项作为最终识别结果。

## 3. 代码填空实现核心参考

```python
# 1. 模型加载
ort_session = onnxruntime.InferenceSession('mnist.onnx')

# 2. 图像处理
image = Image.open('img_test.png').convert('L')
image = image.resize((28, 28))

# 3. 维度变换 (28,28) -> (1,28,28) -> (1,1,28,28)
image_array = np.array(image, dtype=np.float32) / 255.0
image_array = np.expand_dims(image_array, axis=0) # Channel
image_array = np.expand_dims(image_array, axis=0) # Batch

# 4. 推理
ort_inputs = {ort_session.get_inputs()[0].name: image_array}
ort_outs = ort_session.run(None, ort_inputs)

# 5. 获取预测值
predicted_class = np.argmax(ort_outs[0])
```

## 4. 人机交互最优流程设计建议
- **反馈闭环**：在识别结果下方提供“正确/错误”反馈按钮，增强系统学习潜力。
- **可视化输出**：显示 0-9 所有数字的概率条形图，让用户直观看到模型的“思考过程”。
- **自动分割**：对于多位数字（如 123），系统应支持自动分割并将每个数字单独识别，实现连续书写识别。
