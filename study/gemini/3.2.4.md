# 3.2.4 花朵智能识别系统交互流程设计解析

## 1. 实验目标
本实验旨在实现一个用于绿化管理的花朵自动分类系统。通过加载预训练的深度卷积神经网络模型（ONNX 格式），对输入的花朵图像进行特征提取与分类识别。实验涵盖了图像预处理算法设计、Softmax 概率转换以及推理结果的百分比可视化。

## 2. 关键技术环节

### 2.1 图像中心裁剪 (Center Crop)
由于花朵通常位于图像的中心位置，为了减少背景干扰并满足模型输入的固定尺寸要求，预处理流程采用了“先缩放、后中心裁剪”的策略。这能最大程度保留花朵的核心特征。

### 2.2 归一化与色彩空间
- **归一化参数**：均值 `[0.485, 0.456, 0.406]` 和标准差 `[0.229, 0.224, 0.225]` 是基于 ImageNet 大规模数据集计算得到的标准参数，能让模型在训练和预测时处于相同的数据分布。
- **维度转换**：将图像矩阵从 NumPy 默认的 `(H, W, C)` 转换为 ONNX 推理所需的 `(1, C, H, W)`。

### 2.3 概率可视化与张量索引 (Tensor Indexing)
- **批次访问 (Batch Access)**: 推理引擎返回的 `accuracy` 形状通常为 `(1, N)`，其中 `1` 代表批次大小（Batch Size）。即使只识别单张图，也必须使用索引 `0` 来进入该批次。
- **精准定位 (Predicted Index)**: 通过 `accuracy[0, predicted_idx]` 语法，系统能够从二维张量中准确提取出概率最大的那个浮点数值（例如从 `0.985` 提取）。
- **百分比映射**: 将 `[0, 1]` 区间的原始概率值乘以 `100`，转换为符合人类认知习惯的百分数（如 98.50%），从而直观展示模型识别的“确信度”。

## 3. 代码实现核心参考

```python
# 1. 模型加载
session = ort.InferenceSession('flower-detection.onnx')

# 2. 图像读取与预处理
image = Image.open('flower_test.png').convert('RGB')
processed_image = preprocess_image(image)

# 3. 执行推理
output = session.run([output_name], {input_name: processed_image})[0]

# 4. 后处理解码
accuracy = scipy.special.softmax(output, axis=-1)
predicted_idx = np.argmax(accuracy[0]) # 获取最可能的类别索引

# 5. 核心计算：批次索引 + 概率提取 + 百分比转换
prob_percentage = accuracy[0, predicted_idx] * 100 
predicted_label = labels[predicted_idx]
```

## 4. 人机交互最优流程设计方案 (docx 要点)
- **实时性**：在取景框中实时显示识别动态，并对重点区域进行聚焦，引导用户拍摄更清晰的局部图。
- **信息扩展**：识别成功后不应只显示名字，还应关联花朵的养护知识、花语等百科信息，提升交互深度。
- **容错性**：当光线过暗或花朵残缺导致置信度低于 50% 时，主动提示用户“环境较差，请重试”或“距离太远”，实现智能化的交互引导。
