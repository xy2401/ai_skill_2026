# 2.2.1 智能信用评分Logistic回归模型开发与测试解析

## 1. 实验目标
本实验旨在通过 Python 编程，利用逻辑回归（Logistic Regression）算法构建一个个人信用评分模型。通过对历史金融数据的学习，模型能够根据客户的年龄、收入、负债率、逾期记录等特征，预测其未来两年内是否会出现严重的贷款拖欠行为。

## 2. 数据说明
数据集包含以下关键特征：
- `SeriousDlqin2yrs`: **目标变量**。1 表示有严重拖欠，0 表示没有。
- `RevolvingUtilizationOfUnsecuredLines`: 信用卡及个人信贷额度使用率。
- `age`: 客户年龄。
- `NumberOfTime30-59DaysPastDueNotWorse`: 逾期 30-59 天的次数。
- `DebtRatio`: 负债率。
- `MonthlyIncome`: 月收入。
- `NumberOfOpenCreditLinesAndLoans`: 信贷账户数量。
- `NumberOfTimes90DaysLate`: 逾期 90 天以上的次数。
- `NumberRealEstateLoansOrLines`: 房地产贷款数量。
- `NumberOfTime60-89DaysPastDueNotWorse`: 逾期 60-89 天的次数。
- `NumberOfDependents`: 家属人数。

## 3. 核心步骤详解

### 3.1 数据加载与预处理
首先使用 `pandas` 读取 CSV 数据。在特征选择时，需要剔除与预测无关的 `Unnamed: 0`（索引列）和目标列 `SeriousDlqin2yrs`。

### 3.2 逻辑回归模型构建
逻辑回归是处理二分类问题的经典算法。在训练过程中，我们设置 `max_iter=1000` 以确保模型在梯度下降过程中能够充分收敛。

### 3.3 应对数据不平衡（SMOTE）
在信用风险评估中，违约客户（少数类）通常远少于守信客户（多数类）。如果直接训练，模型会倾向于预测所有人都不会违约。
- **SMOTE 技术**：通过合成新的少数类样本，使训练集在类别上达到平衡，从而显著提升模型对风险客户的识别能力（即提高 Recall 召回率）。
