# 3.2.5 人脸 AI 智能检测系统交互流程设计解析

## 1. 实验目标
本实验旨在实现一个高效的人脸自动检测系统。通过 ONNX Runtime 引擎部署轻量级检测模型，实现对批量图像中人脸位置的精准定位与数量统计。实验重点在于理解目标检测中的图像预处理链路及复杂的后处理逻辑（如 NMS 非极大值抑制）。

## 2. 关键技术环节

### 2.1 图像预处理链路 (Pre-processing Pipeline)
模型对输入的输入有严格的几何与数值规范：
- **颜色空间转换**：OpenCV 默认使用 BGR 格式，而模型通常是在 RGB 空间下训练的，因此需使用 `cv2.cvtColor` 进行转换。
- **特定归一化 (Standardization)**：采用减去均值 127 并除以 128 的策略，将数据映射到 `[-1, 1]` 区间，降低光照变化对识别的影响。
- **NCHW 张量构造**：利用 `np.transpose` 将通道维度提前，并用 `np.expand_dims` 增加批次维度，构造出形状为 `(1, 3, 240, 320)` 的标准输入张量。

### 2.2 推理与预测逻辑
- **ONNX 推理**：`ort_session.run` 负责执行前向传播，返回包含边界框坐标（Boxes）和各类别置信度（Confidences）的数组。
- **后处理 (NMS)**：目标检测模型往往会针对同一个目标产生多个重叠框。通过 `hard_nms` 算法，系统能根据 IOU（交并比）剔除冗余，仅保留最准确的检测结果。

## 3. 代码实现核心参考

```python
# 1. 初始化推理会话
ort_session = ort.InferenceSession('version-RFB-320.onnx')
input_name = ort_session.get_inputs()[0].name

# 2. 图像批量读取与缩放
orig_image = cv2.imread(img_path)
image = cv2.resize(cv2.cvtColor(orig_image, cv2.COLOR_BGR2RGB), (320, 240))

# 3. 归一化与维度扩展
image_mean = np.array([127, 127, 127])
image = (image - image_mean) / 128
image = np.expand_dims(np.transpose(image, [2, 0, 1]), axis=0).astype(np.float32)

# 4. 执行检测推理
confidences, boxes = ort_session.run(None, {input_name: image})

# 5. 后处理与绘图
boxes, labels, probs = predict(orig_image.shape[1], orig_image.shape[0], confidences, boxes, threshold)
cv2.rectangle(orig_image, (box[0], box[1]), (box[2], box[3]), (255, 255, 0), 4)
```

## 4. 人机交互优化方案 (docx 要点)
- **实时监控界面**：设计包含“实时流预览区”、“人流量统计看板”以及“历史抓拍回溯区”的综合布局。
- **参数动态调节**：允许用户在界面上实时拖动滑块调整置信度阈值（Threshold），以适应不同复杂程度的光照场景。
- **智能化导出**：系统应支持生成包含时间戳、检测人数及截图标注的 PDF 检测报告，方便安防存档。
