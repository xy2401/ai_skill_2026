# 3.2.1 图像识别评估系统交互流程设计解析

## 1. 实验目标
本实验旨在实现一个基于 ResNet 深度残差网络的图像识别系统。通过 ONNX Runtime 推理引擎，加载预训练模型对测试图片进行分类识别。实验涵盖了深度学习模型部署的完整链路：模型加载、图像预处理、前向推理以及结果后处理。

## 2. 关键技术概念

### 2.1 ONNX Runtime
ONNX 是一种开放模型交换格式。`onnxruntime` 是专门为 ONNX 模型优化的推理库，能够跨硬件平台（CPU/GPU）实现高性能计算。

### 2.2 图像预处理 (Image Preprocessing)
为了匹配 ResNet 模型的训练环境，输入图片必须经过以下标准化步骤：
- **缩放与中心裁剪**：确保输入尺寸为固定的 224x224 像素。
- **色彩空间归一化**：将 0-255 的像素值转化为 0.0-1.0，并应用特定的均值 (Mean) 和标准差 (Std) 进行标准化，使模型输入分布稳定。
- **维度重排 (Transpose)**：将图片的 `HWC` (高、宽、通道) 顺序转换为模型要求的 `CHW` (通道、高、宽) 顺序。

### 2.3 结果后处理 (Softmax & Top-K)
- **Softmax 函数**：模型最后一层的原始输出（Logits）没有直观含义。通过 Softmax 变换，将其转化为总和为 1 的概率分布。
- **Top-5 筛选**：在成百上千个类别中，通过排序获取概率最高的前 5 位，这比仅输出单一类别更具容错性和信息量。

## 3. 代码填空实现核心参考

```python
# 1. 模型加载
session = ort.InferenceSession('resnet.onnx')

# 2. 图像读取与预处理
image = Image.open('img_test.jpg').convert('RGB')
processed_image = preprocess_image(image)

# 3. 执行推理
# session.run 要求传入输出名称列表和输入字典
output = session.run([output_name], {input_name: processed_image})[0]

# 4. 概率转换与排序
probabilities = scipy.special.softmax(output, axis=-1)
# 索引排序：利用 argsort 获取最高概率的前五个索引
top5_idx = np.argsort(probabilities[0])[-5:][::-1]
top5_prob = probabilities[0][top5_idx]
```

## 4. 人机交互最优方式设计 (docx 要点)
- **透明性**：显示 Top-5 类别及对应概率，而不仅仅是最终结果，让用户了解模型的确定程度。
- **引导性**：当最高置信度较低时，提示用户“拍摄角度可能不佳”或“特征不明显”。
- **闭环性**：提供“报错/修正”机制，收集人工纠偏数据，为模型的迭代升级提供高质量样本。
