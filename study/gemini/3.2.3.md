# 3.2.3 面部表情识别系统交互流程设计解析

## 1. 实验目标
本实验旨在构建一个端到端的面部表情识别（Facial Expression Recognition）应用。通过加载基于 FER+ 数据集训练的 ONNX 模型，对输入的人脸图像进行情绪分类（如快乐、惊讶、愤怒等）。实验重点考察了图像灰度化预处理、模型输入张量构造以及输出结果的逻辑解码。

## 2. 关键技术环节

### 2.1 图像预处理 (Pre-processing)
与通用的物体识别不同，面部表情识别模型（尤其是 FER+ 变体）通常对输入有特定要求：
- **灰度化 (L 模式)**：表情识别主要依赖肌肉的拉伸和阴影变化（纹理特征），而非色彩信息，因此使用灰度图可减少计算量并过滤环境光颜色干扰。
- **空间一致性**：必须精准调整图像尺寸至 64x64 像素。
- **4D 张量构造**：使用 `np.expand_dims` 连续增加 Batch 和 Channel 维度，最终形成 `(1, 1, 64, 64)` 形状，确保与模型输入层的 Gate（门控）完全匹配。

### 2.2 结果解析逻辑
- **Argmax 提取**：模型对每一个情感类别都会输出一个“原始分值”。我们利用 `np.argmax` 找到这些分值中最大值所在的下标（即索引）。
- **映射转换**：通过预先定义好的字典（如 `0 -> neutral`），将枯燥的数字索引还原为具有人类语义的情绪标签。

## 3. 代码实现核心参考

```python
# 1. 情感标签映射
emotion_table = {'neutral':0, 'happiness':1, 'surprise':2, 'sadness':3, 'anger':4, 'disgust':5, 'fear':6, 'contempt':7}
emotion_list = list(emotion_table.keys()) # 转为列表方便索引访问

# 2. 模型初始化
ort_session = ort.InferenceSession('emotion-ferplus.onnx')

# 3. 图像预处理
input_data = preprocess('img_test.png')

# 4. 执行识别推理
# ort_session.run(输出节点名, 输入字典)
ort_outs = ort_session.run(None, {ort_session.get_inputs()[0].name: input_data})

# 5. 解码输出
predicted_label = np.argmax(ort_outs[0])
predicted_emotion = emotion_list[predicted_label]
```

## 4. 人机交互最优设计方案 (docx 要点)
- **动感反馈**：将识别出的情绪转化为动态的卡通表情包同步显示，增加交互的趣味性。
- **时序分析**：系统不仅应识别单帧，还应记录一段时间内的情绪变化趋势（如：用户在观看广告时的情绪曲线），提供深度洞察。
- **环境自适应**：自动根据环境光线调整输入图像的对比度，确保在弱光环境下依然能准确捕捉面部肌肉的细微变化。
